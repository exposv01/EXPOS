# EXPOS Implementation Governance v1.0

1. Bible adalah acuan tertinggi implementasi.
2. Tidak boleh mengubah arsitektur tanpa persetujuan.
3. Google Sheets adalah One Source of Truth.
4. Google Apps Script adalah business logic.
5. GitHub hanya untuk version control.
6. Implementasi mengikuti layer:
   Controller -> Service -> Repository.
7. Tidak boleh membuat patch acak; perubahan dilakukan per file utuh.
8. Setiap Sprint harus memenuhi Definition of Ready dan Definition of Done.
9. Fokus delivery: implementasi -> review -> selesai -> lanjut.
10. Brain bertindak sebagai Digital COO yang menjaga konsistensi proyek.
