# UI-009 — Absensi Mobile Form

Tanggal delivery: 2026-07-04  
Sprint: Sprint 4 Part 2  
Status: Implemented in GitHub  
Repository: exposv01/EXPOS

## Objective

Membuat halaman form mobile-first untuk Absensi sebagai lanjutan dari UI-008 Report Problem.

UI-009 berfungsi sebagai presentation layer untuk input absensi karyawan cabang.

## Bible Compliance

Acuan implementasi:

- Google Sheets tetap menjadi database operasional.
- Google Apps Script tetap menjadi business logic dan automation layer.
- GitHub hanya menjadi source control untuk source code.
- UI hanya presentation layer.
- Workflow submit tetap melalui Controller/Service.
- UI tidak membaca atau menulis Sheet langsung.
- UI tidak menyimpan file ke Drive secara langsung.
- Identitas user final tetap dari Google Session dan Master_User.
- Nama karyawan di form hanya temporary UI field sampai backend session/role disambungkan penuh.
- Tidak membuat database baru.
- Tidak menambah payroll, inventory, approval, atau dashboard kompleks.

## Scope UI-009

### Included

1. Mobile shell dengan max-width 390px.
2. Header Absensi.
3. Status card dengan greeting dan jam WIB.
4. Tipe absensi:
   - Masuk / Check In
   - Pulang / Check Out
5. Input cabang:
   - CBN
   - ARH
   - SLW
6. Input shift:
   - Shift 1
   - Shift 2
   - Middle
   - Full Day
7. Input nama karyawan sebagai temporary UI field.
8. Input catatan opsional.
9. Upload foto selfie/bukti hadir sebagai UI placeholder.
10. Client-side validation ringan.
11. Hook aman ke Apps Script Controller:
   - `submitAttendance(payload)`
   - `openExposModule('home')`
12. Fallback aman jika controller belum tersedia.

### Excluded

1. Direct write ke Sheet.
2. Direct upload ke Drive.
3. Payroll calculation.
4. Approval workflow.
5. Shift scheduler logic.
6. Geo-location enforcement.
7. Face recognition.
8. Admin analytics.
