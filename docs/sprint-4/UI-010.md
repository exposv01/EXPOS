# UI-010 — Kasbon Mobile Form

Tanggal delivery: 2026-07-04  
Sprint: Sprint 4 Part 2  
Status: Implemented in GitHub  
Repository: exposv01/EXPOS

## Objective

Membuat halaman form mobile-first untuk pengajuan Kasbon sebagai lanjutan dari UI-009 Absensi.

UI-010 berfungsi sebagai presentation layer untuk input pengajuan kasbon karyawan cabang.

## Bible Compliance

Acuan implementasi:

- Google Sheets tetap menjadi database operasional.
- Google Apps Script tetap menjadi business logic dan automation layer.
- GitHub hanya menjadi source control untuk source code.
- UI hanya presentation layer.
- Workflow submit tetap melalui Controller/Service.
- UI tidak membaca atau menulis Sheet langsung.
- UI tidak membuat approval workflow.
- UI tidak menghitung payroll atau potongan gaji.
- Identitas user final tetap dari Google Session dan Master_User.
- Nama karyawan di form hanya temporary UI field sampai backend session/role disambungkan penuh.
- Tidak membuat database baru.
- Tidak menambah inventory, analytics, atau dashboard kompleks.

## Scope UI-010

### Included

1. Mobile shell dengan max-width 390px.
2. Header Kasbon.
3. Info card instruksi pengajuan.
4. Input cabang:
   - CBN
   - ARH
   - SLW
5. Input tanggal pengajuan.
6. Input nama karyawan sebagai temporary UI field.
7. Input nominal kasbon dengan preview Rupiah.
8. Input alasan pengajuan.
9. Input rencana pengembalian opsional:
   - Potong gaji bulan ini
   - Potong gaji bulan depan
   - Cicilan 2x
   - Cicilan 3x
10. Client-side validation ringan.
11. Hook aman ke Apps Script Controller:
   - `submitKasbon(payload)`
   - `openExposModule('home')`
12. Fallback aman jika controller belum tersedia.

### Excluded

1. Direct write ke Sheet.
2. Payroll calculation.
3. Approval workflow.
4. Auto limit kasbon.
5. Pencairan kasbon.
6. Rekap kasbon admin.
7. Notification workflow.
8. Admin analytics.

## Controller Contract

UI-010 siap disambungkan ke Apps Script function berikut:

```javascript
function submitKasbon(payload) {
  // UI payload example:
  // {
  //   branch: 'CBN',
  //   requestDate: '2026-07-04',
  //   employeeName: 'Nama karyawan',
  //   amount: 250000,
  //   reason: 'Kebutuhan mendesak',
  //   repaymentPlan: 'Potong gaji bulan ini',
  //   clientTimestamp: '2026-07-04T...Z'
  // }
  // Backend must:
  // - validate payload again
  // - resolve user identity from Google Session
  // - resolve role/branch from Master_User
  // - pass workflow to KasbonService
  // - persist via Repository/Helper only
  // - avoid payroll calculation in this UI task
}

function openExposModule(moduleName) {
  // UI-010 uses openExposModule('home') for back navigation.
}
```
