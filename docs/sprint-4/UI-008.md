# UI-008 — Report Problem Mobile Form

Tanggal delivery: 2026-07-04  
Sprint: Sprint 4 Part 2  
Status: Implemented in GitHub  
Repository: exposv01/EXPOS

## Objective

Membuat halaman form mobile-first untuk Report Problem sebagai lanjutan dari UI-007 Home Dashboard.

UI-008 berfungsi sebagai presentation layer untuk input laporan masalah operasional cabang.

## Bible Compliance

Acuan implementasi:

- Google Sheets tetap menjadi database operasional.
- Google Apps Script tetap menjadi business logic dan automation layer.
- GitHub hanya menjadi source control untuk source code.
- UI hanya presentation layer.
- Workflow submit tetap melalui Controller/Service.
- UI tidak membaca atau menulis Sheet langsung.
- UI tidak menyimpan file ke Drive secara langsung.
- Identitas user tidak diambil dari input manual sebagai source of truth; backend tetap harus membaca Google Session dan Master_User.
- Tidak membuat database baru.
- Tidak menambah approval, inventory, payroll, atau dashboard kompleks.

## Scope UI-008

### Included

1. Mobile shell dengan max-width 390px.
2. Header Report Problem.
3. Info card instruksi laporan.
4. Input cabang:
   - CBN
   - ARH
   - SLW
5. Input kategori:
   - Konsol
   - TV / Monitor
   - Controller
   - Jaringan
   - Fasilitas
   - Lainnya
6. Input judul masalah.
7. Pilihan prioritas:
   - Low
   - Medium
   - High
8. Input deskripsi masalah.
9. Input nama pelapor sebagai temporary UI field.
10. Upload foto pendukung sebagai UI placeholder.
11. Client-side validation ringan.
12. Hook aman ke Apps Script Controller:
   - `submitReportProblem(payload)`
   - `openExposModule('home')`
13. Fallback aman jika controller belum tersedia.

### Excluded

1. Direct write ke Sheet.
2. Direct upload ke Drive.
3. Approval workflow.
4. Notification workflow.
5. Problem assignment.
6. Problem closing workflow.
7. Bell notification aktif.
8. Admin analytics.
