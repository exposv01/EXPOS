# UI-011 — Pengajuan Izin Mobile Form

Tanggal delivery: 2026-07-04  
Sprint: Sprint 4 Part 2  
Status: Implemented in GitHub  
Repository: exposv01/EXPOS

## Objective

Membuat halaman form mobile-first untuk Pengajuan Izin sebagai lanjutan dari UI-010 Kasbon.

UI-011 berfungsi sebagai presentation layer untuk input pengajuan izin karyawan cabang.

## Bible Compliance

Acuan implementasi:

- Google Sheets tetap menjadi database operasional.
- Google Apps Script tetap menjadi business logic dan automation layer.
- GitHub hanya menjadi source control untuk source code.
- UI hanya presentation layer.
- Workflow submit tetap melalui Controller/Service.
- UI tidak membaca atau menulis Sheet langsung.
- UI tidak menyimpan file ke Drive secara langsung.
- UI tidak membuat approval workflow.
- UI tidak menghitung payroll, cuti, atau potongan gaji.
- Identitas user final tetap dari Google Session dan Master_User.
- Nama karyawan di form hanya temporary UI field sampai backend session/role disambungkan penuh.
- Tidak membuat database baru.
- Tidak menambah inventory, analytics, atau dashboard kompleks.

## Scope UI-011

### Included

1. Mobile shell dengan max-width 390px.
2. Header Pengajuan Izin.
3. Info card instruksi pengajuan.
4. Input cabang:
   - CBN
   - ARH
   - SLW
5. Input jenis izin:
   - Sakit
   - Izin Pribadi
   - Keluarga
   - Darurat
   - Lainnya
6. Input nama karyawan sebagai temporary UI field.
7. Input tanggal mulai.
8. Input tanggal selesai.
9. Preview durasi izin dalam hari.
10. Input alasan izin.
11. Upload lampiran pendukung sebagai UI placeholder.
12. Client-side validation ringan.
13. Hook aman ke Apps Script Controller:
   - `submitIzin(payload)`
   - `openExposModule('home')`
14. Fallback aman jika controller belum tersedia.

### Excluded

1. Direct write ke Sheet.
2. Direct upload ke Drive.
3. Approval workflow.
4. Payroll calculation.
5. Auto saldo cuti/izin.
6. Auto schedule replacement.
7. Notification workflow.
8. Admin analytics.

## Controller Contract

UI-011 siap disambungkan ke Apps Script function berikut:

```javascript
function submitIzin(payload) {
  // UI payload example:
  // {
  //   branch: 'CBN',
  //   leaveType: 'Sakit',
  //   employeeName: 'Nama karyawan',
  //   startDate: '2026-07-04',
  //   endDate: '2026-07-04',
  //   durationDays: 1,
  //   reason: 'Sakit demam',
  //   hasAttachment: false,
  //   clientTimestamp: '2026-07-04T...Z'
  // }
  // Backend must:
  // - validate payload again
  // - resolve user identity from Google Session
  // - resolve role/branch from Master_User
  // - pass workflow to IzinService
  // - persist via Repository/Helper only
  // - optionally handle attachment through Drive service layer
  // - avoid payroll/cuti balance calculation in this UI task
}

function openExposModule(moduleName) {
  // UI-011 uses openExposModule('home') for back navigation.
}
```
