# UI-012 — Rekap Absensi Mobile Page

Tanggal delivery: 2026-07-04  
Sprint: Sprint 4 Part 2  
Status: Implemented in GitHub  
Repository: exposv01/EXPOS

## Objective

Membuat halaman mobile-first untuk Rekap Absensi sebagai lanjutan dari UI-011 Pengajuan Izin.

UI-012 adalah presentation layer read-only untuk melihat ringkasan dan detail absensi karyawan cabang.

## Bible Compliance

- Google Sheets tetap menjadi database operasional.
- Google Apps Script tetap menjadi business logic dan automation layer.
- GitHub hanya menjadi source control untuk source code.
- UI hanya presentation layer.
- Data rekap tetap dibaca melalui Controller/Service.
- UI tidak membaca Sheet langsung.
- UI tidak menulis Sheet langsung.
- UI tidak membuat dashboard kompleks.
- UI tidak melakukan analytics payroll.
- Identitas dan akses data tetap divalidasi backend dari Google Session dan Master_User.
- Tidak membuat database baru.

## Scope UI-012

### Included

1. Mobile shell dengan max-width 390px.
2. Header Rekap Absensi.
3. Summary card periode: Hadir, Izin, Telat/Alpha.
4. Filter cabang: Semua, CBN, ARH, SLW.
5. Filter periode bulan.
6. List detail absensi read-only.
7. Empty state jika data belum tersedia.
8. Hook aman ke Apps Script Controller: getAttendanceRecap(filter) dan openExposModule('home').
9. Fallback aman jika controller belum tersedia.

### Excluded

1. Direct read dari Sheet.
2. Direct write ke Sheet.
3. Export Excel/PDF.
4. Payroll calculation.
5. Approval workflow.
6. Admin analytics kompleks.
7. Editing absensi.
8. Delete/correction workflow.

## Controller Contract

UI-012 siap disambungkan ke Apps Script function getAttendanceRecap(filter).

Filter dari UI berisi branch, period, dan clientTimestamp.

Backend wajib:

- Validate filter ulang.
- Resolve user identity dari Google Session.
- Resolve role dan branch access dari Master_User.
- Pass workflow ke AttendanceService atau AbsensiService.
- Read data via Repository/Helper only.
- Return safe read-only data untuk summary dan items.

Expected response shape:

- summary.hadir
- summary.izin
- summary.telatAlpha
- items[].employeeName
- items[].branch
- items[].date
- items[].shift
- items[].checkIn
- items[].checkOut
- items[].status
